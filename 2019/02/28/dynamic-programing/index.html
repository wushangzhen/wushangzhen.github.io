Dynamic Programming

Posted on 2019-03-01 | In Algorithm

Introduction

动态规划是必须要掌握的算法，去年上过关于动态规划的专题班，但是没有更深一步的总结，这一轮刷题的时候还是要深入的总结一下动态规划的常见题型，常见的动态规划算法有两种，一种top down主要代表就是记忆化搜索，另一种是bottomup这种其实就是递推公式类型的，先把LeetCode上关于动态规划的常见题型总结一下。

Longest Palindromic Substring

Longest Palindromic Substring
区间型动态规划,空间复杂度O(n^2), 也可以用字符串扩展的方法实现O(1)空间复杂度

dp

字符串扩展

Longest Palindromic Subsequence

Longest Palindromic Subsequence
区间型动态规划, 这题求的是subsequence，代表这个区间内最长的子序列，可以不以i结尾。

Regular Expression Matching

Regular Expression Matching
双序列型动态规划，这个题属于记忆化搜索，也是dp的一种，我们要做的事，建立一个visited数组和results数组记录结构，用两根指针来记录遍历位置，注意终止条件, 建议做十遍。

Wildcard Matching

Wildcard Matching
这个题比上一个题要简单一点，但是大致思路是一样的，记忆化搜索，bugfree

Longest Valid Parentheses

Longest Valid Parentheses
序列型动态规划，注意递推条件，遍历，如果遇到 ( 或者 j = i - dp[i - 1] - 1 < 0 或者 sc[j] == )都continue掉，代表以i-1结尾的字符不能构成valid parenthesis。其他的都是dp[i] = 2 + dp[i - 1] + dp[j];

Edit Distance

Edit Distance
这个题是双序列型动态规划, 三种情况分别算，初始条件搞清楚。

可以用滚动数组优化空间复杂度O(n)

Scramble String

Scramble String
记忆化搜索，分段递归， 但是这个方法效率很低，下次可以看一个比较快的方法。

Decode Ways

Decode Ways
经典题，有很多corner case， 比如100 230等等，主要是以0相关的。如果遍历到这个数为0，需要向前看一位，如果这个位==1或者==2，就是合法的，否则continue;

Unique Binary Search Trees

Unique Binary Search Trees
卡特兰树，参考

Interleaving String

Interleaving String
这个题类似像regular expression match可以采用记忆化搜索top-down的方法，每一个位置枚举可能出现的值，类似序列型动态规划，建议十遍。

Distinct Subsequences

Distinct Subsequences
和上题类似，是区间型型动态规划，并不好想。

Best Time to Buy and Sell Stock III

Best Time to Buy and Sell Stock III
关键是五种状态需要搞清楚，还有限制条件。

Best Time to Buy and Sell Stock IV

Best Time to Buy and Sell Stock IV
这个是sell stock 2 和 3的结合，两种情况分状态分别讨论就好。

Palindrome Partitioning II

Palindrome Partitioning II
字符串扩展方法判断palindrome, 然后序列型dp求解。

Burst Balloons

Burst Balloons
感觉这个题top-down + memoization更加好做一点，根据相应的范围进行recursion，还是挺巧妙的

Paint House II

Paint House II
这个题很容易写出O(nk^2)的解法，想要写出O(nk)的解法就需要维持四个临时变量，minCost, secondCost, minColor, secondColor, 在遍历的过程中不断进行更新，最后输出mincost即可

Word Break

Word Break
这个也要用记忆化搜索来通过大的数据集，利用Boolean数组可以存三个值的特性，来进行记录。

Word Break II

Word Break II
记忆化搜索，注意边界条件和初始条件，用trim()可以省去判断边界条件的情况。

Dungeon Game

Dungeon Game
坐标型动态规划，关键点在于要扩展2darray，注意初始化的值，以及从尾部开始进行递归，这样比较简单。

Integer Break

Integer Break
这个题可以用O(n^2)的动态规划，每次算的时候需要进行dp[i]与i的大小比较，因为dp[i]是不包括本身i的值的，但是算大数的时候可能需要i本身的值，这个事关键的点。

House Robber II

House Robber II
这个题和I不大一样，需要把环分为两条不同的链，规定了起始和结尾部分，这样就可以调用I的dp方法，注意值的初始化和返回值应该返回什么。

House Robber III

House Robber III
这个题是依托树的变种，记忆化搜索，递归进行查询，还是挺好的题

Ugly Number II

Ugly Number II
我一开始做的是用priorityqueue+set去重判断，这样的话会比较慢，这个可以类似mergesort，每个factor都维持一个index和一个factor，factor用来给ugly[]赋值，各自的index用来找更新下一个值。

Paint Fence

Paint Fence
这个题要求不能超过两个fence颜色一样，可以维持两个变量，same表示当前与之前颜色一样，diff表示当前与之前颜色不一样，不断更新same和diff的值，diff= same (k - 1) 有k-1种可能 + diff (k - 1) 颜色与之前不一样也有k-1种可能，same就要更新为原来diff代表一样。
